## Быстрые фильтры (SmartChip) — как добавить новый чип

Этот файл кратко описывает, как устроена панель быстрых фильтров и что нужно сделать, чтобы добавить новый смарт‑чип.

### Основные файлы
- `src/components/QuickFiltersBar/QuickFiltersBar.tsx` — конфиг чипов и рендер панели
- `src/components/Chip/SmartChip.tsx` — универсальный компонент смарт‑чипа
- `src/components/Chip/SmartChip.module.css` — стили смарт‑чипа (светлая/тёмная темы)
- `src/utils/aiFilterMapper.ts` — маппинг ответа ИИ в стор фильтров

### Архитектурная идея
Панель собирается на основе конфигурации. Чтобы добавить новый чип, не нужно трогать много мест — достаточно:
1) расширить тип идентификаторов и порядок чипов,
2) добавить описание чипа в общий конфиг `UI_CHIPS`.

### Как добавить новый чип
1. Откройте `QuickFiltersBar.tsx` и обновите:
   - `UIChipId` — добавьте новый идентификатор, например `'cookTime'`.
   - `UI_CHIPS_ORDER` — добавьте идентификатор в нужное место списка.
   - `UI_CHIPS` — добавьте объект с полями:
     - `id` — идентификатор из `UIChipId`
     - `label` — подпись чипа
     - `render(ctx)` — функция, возвращающая `SmartChip` нужного типа (`label` | `range` | `select`).

2. Если чип должен автоматически появляться при установке значений (например, из ИИ), добавьте проверку в единый эффект авто‑показа в `QuickFiltersBar.tsx` (поиск по комментариям «Автопоказ чипов на основе значений фильтров»).

3. Если значения приходят из ИИ‑поиска, добавьте маппинг в `mergeAiFilterIntoFilters` (`src/utils/aiFilterMapper.ts`), чтобы преобразовать серверный формат в формат стора (`RecipeFilters`). Панель подтянет значения автоматически из `filters`.

### Виды смарт‑чипов
Компонент `SmartChip` поддерживает 3 вида:
- `label` — статическая метка (только текст)
  - пропсы: `text`, `title`
- `range` — диапазон (от/до)
  - пропсы: `from`, `to`, `placeholderFrom`, `placeholderTo`, `toOnly` (если `true`, показывается только поле «до»), `onChange`
  - значения контролируемые (через `value`), отображаются из пропсов
- `select` — выпадающий список
  - пропсы: `value`, `options: { value, label }[]`, `onChange`

Общие пропсы для всех видов: `active` (подсветка активного), `removeMode` (подсветка режима удаления/скрытия в режиме редактирования), `title`.

### Пример: чип «Порции» (диапазон без «от»)
В `UI_CHIPS`:
```tsx
servings: {
  id: 'servings',
  label: 'Порции',
  render: ({ filters, removeMode, active }) => (
    <SmartChip
      kind="range"
      title="Порции"
      to={filters.servings?.max}
      placeholderTo="до"
      toOnly
      removeMode={removeMode}
      active={active}
      onChange={() => { /* заглушка: не фильтруем */ }}
    />
  )
}
```
Если из ИИ приходит `minUnitCount/maxUnitCount`, в `aiFilterMapper` они конвертируются в `filters.servings.min/max`, и чип отобразит значения.

### Сохранение состояния
Панель сохраняет набор видимых чипов/активных чипов в `localStorage`:
- `quickFilters.visibleChips` — какие чипы показаны на панели
- `quickFilters.activeChips` — какие чипы помечены активными (визуально)
Есть совместимость со старыми ключами для чипа «Калории».

### Режим редактирования
Кнопка «+ Добавить свой фильтр» включает режим редактирования:
- клик по чипу переносит его в скрытую зону
- клик по элементу в меню «добавить» возвращает чип в активную зону
- `removeMode` включается внутри `SmartChip` для наглядной подсветки

### Темная тема и стили
Стили для чипов адаптированы под тёмную тему в `SmartChip.module.css`:
- классы: `.active`, `.removeMode`, `:hover`, `:active`
- тёмные токены и контраст текста уже настроены

### Нюансы и рекомендации
- Значения в чипах — контролируемые (используем `value`), поэтому просто передавайте актуальные значения из `filters`.
- Для новых чипов старайтесь выбирать краткие `id` и добавлять их в `UI_CHIPS_ORDER` — порядок там определяет расположение на панели.
- Если для нового чипа нужны дополнительные автоматические условия показа при изменении `filters`, добавьте проверку в общий эффект авто‑показа.
- Сейчас панель работает в режиме «заглушки» — `onChange` у чипов только визуальный. Для реальной фильтрации пробрасывайте события в `onChange` панели и диспатчите изменения в стор.
