### Руководство: как добавлять новые подвкладки в админ-панели

Ниже краткая инструкция по добавлению новых подвкладок (subtabs) в разделе админ-панели, с переиспользованием общих компонентов.

### 1) Где находятся подвкладки
- Основная конфигурация разделов админ-панели — в `src/pages/Admin/AdminPage.tsx` (см. массив `tabs`).
- Для раздела "Продукты" используется массив `subtabs`, по аналогии с вкладками "Теги" и "Кухни":
  - Пример уже подключённых подвкладок: `products:create`, `products:list`, `products:suggest`.

### 2) Добавление новой подвкладки
1. Создайте компонент подвкладки в каталоге раздела, например:
   - `src/pages/Admin/components/ProductListSubTab/ProductListSubTab.tsx`
   - Структура и подход аналогичны уже существующим подвкладкам в тегах/кухнях
2. Подключите новый компонент в `AdminPage.tsx`, добавив объект в `subtabs` нужного раздела:
```tsx
{
  id: 'products:new',
  label: 'Новая',
  title: 'Новая подвкладка',
  content: <YourNewSubTabComponent />
}
```

### 3) Переиспользование общих компонентов
- Табличный вывод c инлайн-редактированием: `src/pages/Admin/components/EditableTable/EditableTable.tsx`
  - Принимает `rows: {id: string; name: string}[]`, `editing` (map id→черновик имени), `onSave`, `loading`, `emptyText`
  - Поддерживает опцию `enableCopyId` и колбэк `onCopyId`
- Компонент табов (вложенная навигация): `src/components/Tabs/Tabs.tsx`
  - Используйте для переключения между подвкладками (см. примеры в "Теги" и "Кухни")
- Сервисы и авторизация:
  - Базовый URL и axios-инстанс: `src/services/api.ts`
  - Авторизованные запросы: `src/services/auth.ts` (`authorizedFetch`, `getAuthHeaders`)
  - Сервис продуктов: `src/services/productsService.ts` (пример `getAllProducts()`, `createProduct()`)

### 4) Пример простой подвкладки со списком
Создайте компонент, который:
- Загружает данные через сервис (например, `productsService.getAllProducts()`)
- Мапит в `EditableRow[]` (`{ id, name }`)
- Отдаёт в `EditableTable`

Минимальные состояния:
- `loading` — отображение статуса загрузки
- `error` (опционально) — сообщение об ошибке
- `editing` — локальные черновики значений ячеек

### 5) Рекомендации по стилю
- Используйте те же CSS-модули и классы-обёртки, что и у соседних подвкладок (карточки, заголовок)
- Соблюдайте типы из `src/types/*` и не дублируйте типы, уже описанные в проекте

### 6) Чек-лист перед коммитом
- Сборка (`npm run build`) и линтер — без ошибок
- Новая подвкладка видна в UI, переключение работает, заглушки для ошибок/пустого списка предусмотрены
