## Epic 5. Discover / Tinder-лента рецептов (`/discover`)

### 5.1 Route & Shell
* Добавить маршрут `/discover` → `DiscoverPage` с обёрткой для колоды карточек

### 5.2 SwipeDeck-компонент
* Реализовать SwipeDeck на базе Framer Motion или React Spring
* Каждый рецепт — почти полноэкранная карточка с обложкой, заголовком и анонсом
* Жесты: свайп влево — пропустить, вправо — сохранить в избранное

### 5.3 Объёмный текст ингредиентов
* Внутри карточки — скроллируемый блок ингредиентов (до 50% высоты карточки)
* При клике по этому блоку — разворачивать полный список ингредиентов и краткий preview шагов

### 5.4 Индикация недостающих продуктов
* В клиентском сторе (Context/Redux) хранить fridge
* Для каждой карточки считать missing = recipe.ingredients – fridge.ingredients и сразу показывать бейдж «Не хватает N» в углу
* Добавить рядом иконку предупреждения, цвет которой зависит от количества недостающих (например, >3 — красный)

### 5.5 Моментальный отклик и Lazy-load
* Любое изменение в холодильнике сразу пересчитывает бейджи без запросов на сервер
* Подгружать новые рецепты по мере свайпов (infinite fetch)

### 5.6 Опционально для оптимизации UX
* Кнопка «Детали» на карточке для разворачивания полной инструкции прямо в ленте
* Пульсирующая анимация карточки, если недостаёт более 3 ингредиентов

## Архитектурные принципы

### Структура компонентов
```

  ComponentName/
    ComponentName.tsx
    ComponentName.module.css

```

- Использование CSS Custom Properties для цветов, отступов, размеров
- Поддержка светлой/тёмной темы

### TypeScript типизация
- Строгая типизация для всех компонентов
